서비스 개요

"공고일 현재 이전 데이터를 학습 → 사정률 1순위 예측 → 입찰가 산출"까지의 전체 워크플로를 단일 화면에서 처리합니다. 나라장터(G2B) + 국방조달(D2B) 공공데이터를 자동 수집하여 2019~2026년 약 138만 건의 실데이터로 분석합니다.

핵심 흐름 (4단계)

  1. 공고 선택 — 공고화면에서 분석할 공고를 클릭 (또는 검색)
  2. 빈도분석 — Tab1에서 사정률 발생빈도 히스토그램 + 1순위 예측 후보 확인
  3. 갭분석 — Tab2에서 업체 투찰가격 빈공간(갭) 확인 + 확정사정률 결정
  4. 종합분석 — Tab3에서 3가지 방법 상관관계 + 종합 1순위 + 투찰리스트 엑셀 다운로드

3가지 분석 방법 (PDF 3 챕터 11번)

방법설명탭
① 사정률 발생빈도 분석과거 N개월 사정률 히스토그램 → 빈도최대/공백/차이최대 모드별 1순위 도출Tab1
② 업체사정률 갭분석투찰업체들 사이 가격 빈공간(갭)이 큰 위치를 1순위 후보로 도출Tab2
③ ①+② 결합 + 상관관계1·2 결과를 합치도(Agreement)로 종합하여 최종 1순위 + 신뢰구간 산출Tab3

2-1. 빈도분석 탭 (Tab1)

과거 N개월 동일 공종/지역 표본의 사정률을 히스토그램으로 시각화하고, 3가지 모드(A/B/C)로 1순위 후보를 도출합니다.

3가지 모드 (PDF 3 챕터 1)

모드로직적용 시점
A · 빈도최대막대가 가장 큰 값(빈도 1위) → 100% 기준 ±방향으로 순서 지정과거 패턴이 일관될 때
B · 공백막대가 없는 값(0건 구간) → 100% 기준 ±방향의도적 회피 구간 존재 시
C · 차이최대인접 막대 빈도 차이가 가장 큰 위치 → 100% 기준 ±방향전환점 식별 시

세부값 룰 (Detail Rule)

  • 처음값 직후 — 가장 처음 막대 바로 뒤에서 설정
  • 마지막값 직후 — 가장 마지막 막대 바로 뒤에서 설정
  • 간격 최대 — 인접 구간 사이 간격이 가장 큰 위치 (기본값)

예측값 선택 — 막대 ↔ ListBox 양방향 (PDF 4 ④)

히스토그램 막대를 마우스로 클릭하거나, 옆 ListBox에서 정확한 값(99.5/99.6/.../100.5)을 선택할 수 있습니다. 두 방식은 자동 동기화되어 키보드만으로도 사정률 결정이 가능합니다.

4. 마이페이지

4개 카드 구성: 프로필 / 비밀번호 변경 / 예측 설정 (자동 적용) / 분석 조회 이력.

학습 저장 (자동 적용)

종합분석 화면에서 마지막에 사용한 옵션(기간·모드·세부값 룰·조정값)을 "학습 저장" 버튼으로 저장하면, 다음 공고 진입 시 동일 옵션이 자동 적용됩니다 (PDF 3 챕터 1 - 7번).

저장 항목설명
period_months기간 (1/3/6/12/24)
bucket_mode모드 (A 빈도최대 / B 공백 / C 차이최대)
detail_rule세부값 룰 (처음값직후 / 마지막값직후 / 간격최대)
rate_offset결과값 조정값 (-0.3 ~ +0.3)

분석 조회 이력

최근 분석한 공고 목록 + 분석 종류(빈도/갭/종합) + 일자. 클릭 시 해당 분석 화면으로 재진입.

5. 데이터 업로드

과거 입찰 데이터(CSV/Excel)를 업로드하면 분석 표본에 즉시 반영됩니다.

필수 컬럼 (헤더 일치 필수)

공고번호 공고명 발주기관 기초금액 사정률 낙찰금액 개찰일

· 사정률은 % 단위 (예: 99.45)
· 금액은 원 단위 (예: 1,000,000,000)
· 개찰일 형식: YYYY-MM-DD

업로드 절차

  1. "샘플 CSV 다운로드" 버튼으로 템플릿 받기
  2. 엑셀에서 데이터 입력 후 저장
  3. "파일 드래그 또는 클릭" 영역에 파일 드롭
  4. 업로드 결과 토스트 + 하단 이력 테이블에 기록

관리자 콘솔

관리자 권한(role=admin) 사용자에게만 노출. 좌측 사이드바에 별도 빨간 테마 적용.

데이터 수집 + 진행률 (PDF 4 ①)

대용량 7년치 데이터 다운로드가 중단되어도 마지막 페이지부터 자동 이어받기. 관리자 화면 톱바에 실시간 진행률(예: 수집 12%)이 5초마다 자동 갱신됩니다.

진행 상태 표시

  • 톱바 뱃지: 진행 중일 때만 노란색 뱃지 노출 (관리자 전용)
  • 진행 카드: 데이터 수집 페이지 본문에 페이지/% + 마지막 커서일자
  • 체크포인트: cat_idx:win_idx:page 좌표로 정확한 재개 보장

FAQ — 자주 묻는 질문

분석 결과가 비어 있어요

해당 발주처/지역/공종의 표본이 부족할 수 있습니다. "기관범위 → 자동확장"으로 변경하거나 카테고리 필터를 "전체"로 넓혀보세요. Tab2의 "직전 동일공종 우선" 토글도 도움이 됩니다.

첫 분석 호출이 5초 이상 걸려요

분석 캐시 미적중 시 첫 호출은 3~5초 소요됩니다. 두 번째 호출부터는 50ms 이내로 빠릅니다 (24시간 캐시 유지).

학습 저장 후 다음 공고에 자동 적용되지 않아요

마이페이지 → 예측 설정에서 저장된 옵션이 표시되는지 확인하세요. 다른 사용자가 동시 사용 중이면 본인 계정에만 저장됩니다.

관리자 메뉴가 안 보여요

현재 데모 계정(client@demo.com)은 read-only 권한입니다. 관리자 메뉴는 role=admin 사용자에게만 노출됩니다.

엑셀 다운로드 파일이 깨져요

엑셀 2010 이상에서 정상 열림. 파일 형식은 .xlsx입니다. 한글 폰트 문제 시 시스템 한글 폰트(맑은 고딕 등)가 설치되어 있는지 확인하세요.

데모 URL이 접속이 안 돼요

Cloudflare Tunnel 임시 URL은 서버 재시작 시 변경될 수 있습니다. 안내 페이지 (https://ukiim.github.io/bid-analysis-demo/)에서 최신 URL을 확인하세요.

특정 공고가 누락된 것 같아요 (취소·유찰 등)

취소·유찰·폐기·무효 공고는 분석 정확도를 위해 자동 차단됩니다. 차단 패턴: 취소 / 유찰공고 / 폐기공고 / 무효공고 / 입찰취소 / "각 수요기관" 단가계약.
