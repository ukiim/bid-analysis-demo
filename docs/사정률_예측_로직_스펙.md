## 1. 사정률 발생빈도 & 구간 분석 (Tab1)

### 입력
- 기초금액 대비 **사정률 막대 그래프** (분석 기간 내 사정률 분포)
- 100을 중앙 기준으로 +방향 / -방향 양측

### 출력 — 3가지 구간 선택 방식 (각각 산출)

| 방식 | 선정 기준 | 정렬/범위 |
|---|---|---|
| **A** 빈도 최대 | 막대 가장 큰 값 (가장 많이 나온 사정률) | 100 기준 (+/-) 막대 높이순으로 순서 설정 → 범위 지정 |
| **B** 공백 | 막대 없는 값 | 100 기준 (+/-) 순서 → 범위 지정 |
| **C** 차이 최대 | 막대 차이가 가장 큰 값 (인접 차이) | 100 기준 (+/-) 순서 → 범위 지정 |

### 세부 사정률 값 추출 규칙
막대그래프 아래 **표** 에서:
1. 첫 값 바로 뒤
2. 마지막 값 바로 뒤
3. 또는 **간격이 가장 큰 값**

### 옵션
- **분석 기간 (밀어내기식)**: 1개월 / 3개월 / 6개월 / 24개월
- **분류**: 공사 / 용역 / 동일 업종
- **1순위 예측 기간** (리스트박스): 3개월 / 6개월 / 1년 / 3년
- 선택값은 **다음 공고 분석 시 자동 적용** (학습/저장)

## 2. 업체 사정률 분석 (Tab2)

### 기본
- **KBID 포맷** 을 그대로 차용
- 공고명: 공고리스트에서 선택한 값 자동 입력

### 검색 기능
- 발주처
- 예가 변동폭
- 기초금액
- 업종 조건
- 분석 기간

### 인터랙션
- 사정률 발생빈도(Tab1)에서 정의한 **세부 사정률 선택 범위**를 숫자로 표시
- **숫자가 보이게** + **선택 가능** (체크/클릭)
- Tab1 화면과 Tab2 화면에서 선택할 값의 종류를 **구별하여** 숫자로 표시 + 선택 가능
- 1순위 예측 (리스트박스): 3/6개월 / 1년 / 2년 / 3년

## 3. 종합 분석 (Tab3)

### 자동 연계
1. **Tab1 → Tab3 자동 전이**: 사정률 발생빈도에서 선택한 구간이 자동으로 종합분석 화면에 넘어옴
2. **Tab1 → Tab2 적용**: Tab1 구간을 적용하여 Tab2(업체사정률 분석)에서 **세부값** 결정
   - 세부값 규칙: 간격이 가장 큰 곳 → **100 기준 가까운 쪽**
3. 기존 공고 리스트에 다음 컬럼 추가:
   - 예측 사정률
   - 순위
   - **과거 1순위 값** 표시
   - **1순위만 해당되는 공고 추출** 필터

### 분석 기간 (밀어내기식, 공고일 기준)
3개월 / 6개월 / 9개월 / 12개월 / 24개월

### UI 구성
- 1순위 분석 → **선택박스**
- 1순위 결과 → **리스트박스**
- 1순위 결과 = 1순위 분석에 대한 결과값
- 1순위 분석 = 사정률 발생빈도 구간분석 + 종합분석

### 부가 기능
- 모든 분석 결과 **엑셀 저장**
- 사정률 분석값 기반 **투찰 리스트** 작성 + 엑셀 저장
- **10년 이전 데이터 자동/수동 삭제** (현재 5년 정책 → 10년으로 조정 필요)

### 사정률 분석 3가지 방법
| # | 방법 | 출력 |
|---|---|---|
| 1) | 사정률 발생빈도 분석 | 1순위 예측값 |
| 2) | 업체사정률 분석화면 갭(Gap) 분석 | 1순위 예측값 |
| 3) | 사정률 발생빈도 + 업체사정률 발생빈도 분석 | 1순위 예측값 |

→ 위 1)/2)/3)을 각각 분석하여 예측값 1순위 산출
→ 1)/2)/3)을 **종합적으로 연계** 하여 **상관관계 분석** → 종합 1순위 예측값 산출

## 우선순위 (3주차 1차 로직)

### Phase A — 데이터 준비 (1일)
- DATA_RETAIN_YEARS 5 → 10 변경
- BidResult 정합성 점검 (`assessment_rate` NULL 비율)
- 사정률 분포 확인 SQL 작성

### Phase B — Tab1 빈도/구간 (2일)
1. `_rate_histogram(period, category)` — 사정률 막대 그래프 데이터
2. `_compute_rate_buckets(histogram, mode)` — A(빈도최대)/B(공백)/C(차이최대) 3가지 구간 산출
3. `_extract_detail_rate(bucket, table)` — 첫/끝/최대간격 룰
4. 프론트: Tab1 화면에 기간 셀렉트 + 카테고리 필터 + 구간 산출 결과 시각화

### Phase C — Tab2 KBID 갭 (2일)
1. KBID 포맷 레이아웃 (Search 5종 + 그리드)
2. `_company_gap_analysis(filters)` — 갭 분석 쿼리
3. 세부 사정률 선택 인터랙션

### Phase D — Tab3 종합 (2일)
1. Tab1↔Tab3 상태 공유
2. `_predict_top1_rate(method)` × 3가지
3. `_correlate_predictions()` — 종합 1순위
4. 공고 리스트 컬럼 추가 + 1순위 필터

### Phase E — 엑셀 export + 학습 저장 (1일)
1. xlsxwriter 또는 openpyxl 통합
2. `/api/v1/analysis/export?type=...&fmt=xlsx`
3. 투찰 리스트 작성 화면
4. `prediction_settings` 테이블 + 자동 적용 로직

### Phase F — 회귀 + 검수 (1일)
- pytest 신규 분석 함수 단위 테스트 (+10건)
- 브라우저 시각 회귀 (Chrome MCP)
